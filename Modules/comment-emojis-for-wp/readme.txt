Add a lightweight emoji picker to the default WordPress comment textarea so visitors can insert emojis quickly.

== Description ==

Comment Emojis for WP adds a lightweight emoji picker to the default WordPress comment textarea on your site. Visitors can insert emojis into comments quickly, with optional search, recent emoji history, and picker layout controls.

Features include:

* Add emojis directly to the default WordPress comment textarea.
* Insert multiple emojis into a comment without leaving the form.
* Save and display recently selected emojis in the picker.
* Search emojis by name for faster selection.
* Browse emojis by category or scroll through the picker.
* Control picker layout, including picker position, filter position, and search panel position.
* Hide selected interface elements such as search, recent emojis, or skin tone controls when you want a simpler experience.

This plugin follows WordPress coding standards and is designed to stay lightweight for optimal performance.

== Installation ==

= Automatic installation =

1. Log in to your WordPress admin dashboard.
2. Go to Plugins > Add New.
3. Search for Comment Emojis for WP.
4. Click Install Now, then Activate.

= Manual installation =

1. Download the plugin ZIP file.
2. Upload the extracted plugin folder to the /wp-content/plugins/ directory.
3. Go to Plugins in your WordPress admin dashboard.
4. Activate Comment Emojis for WP.

== Setup ==

1. Go to Comment Emojis > Settings in your WordPress admin dashboard.
2. Configure the available options using the provided checkboxes and dropdowns.
3. Save your changes.
4. Open a post with the default WordPress comment form and test the picker.

== Frequently Asked Questions ==

= What does this plugin work with? =

The plugin is designed for the default WordPress comment textarea markup used by standard comment forms.

= Does it work with custom comment forms? =

Not always. If your theme or plugin replaces the default WordPress comment textarea markup, custom integration may be required.

= Can visitors search for emojis? =

Yes. Emoji search is available and can also be disabled from the plugin settings if you prefer a simpler picker.

= Are recent emojis supported? =

Yes. Recently selected emojis can be displayed in the picker, and you can hide that section from the settings page if needed.

= Does it support mobile devices? =

Yes. The picker works on mobile and desktop devices.

= Are there layout controls in the settings? =

Yes. You can control the picker position, filter position, search panel position, and visibility of selected picker features.

= Any database requirement for emoji support? =

Use a utf8mb4 database collation so emoji characters are stored correctly in comments.
